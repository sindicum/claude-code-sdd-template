# フェーズ別 質問パターン集

各フェーズで使える効果的な質問パターン。

---

## Phase 1: Capture 用

### 会話の入り口

漠然としたアイデアの場合:
```
「まず、作りたいものを一言で教えてください。」

「今一番困っていることは何ですか？
 それをどう解決したいですか？」
```

ある程度具体的な場合:
```
「[要約]ということですね。
 これは [ユーザー像] が [場面] で使うものですか？」
```

### 深掘り

```
「その機能を使う典型的なシーンを、
 ステップバイステップで教えてもらえますか？」

「それがうまく動いたとき、
 画面には何が表示されていますか？」

「逆に失敗したとき、何が起きるべきですか？」
```

### スコープ確認

```
「これは大きなシステムの一部ですか？
 それとも独立したツールですか？」

「全部一度に作らなくていいなら、
 まず何が動けばOKですか？」
```

## Phase 2: Explore 用

### ユーザーストーリー引き出し

```
「他にどんな操作をしたいですか？
 優先度は後で決めるので、思いつく限り挙げてください。」

「その作業の前後でユーザーは何をしていますか？
 前工程・後工程がわかると必要な機能が見えます。」

「管理者や別の役割のユーザーはいますか？」
```

### やらないことの明確化

```
「よくある要望として [X] がありますが、
 今回入れますか？入れませんか？」

「将来やりたいけど今回は見送るものは？」

「[類似サービス] との違い・あえて持たない機能は？」
```

### 既存環境の確認

```
「既存のシステムやデータはありますか？
 連携や移行は必要ですか？」

「今はその作業をどうやっていますか？
 Excel？紙？別のツール？」
```

### 規模感の探り方

数値を引き出すには選択肢を提示するのが効果的:

```
NG: 「パフォーマンス要件はありますか？」
    → 漠然としすぎて有効な回答が得られない

OK: 「データは何件くらいを想定していますか？
     数十件？数千件？数万件以上？
     それによってアプローチが変わります。」
```

```
「このデータは誰に見せてOKですか？
 全員共通？ユーザーごとに分離？」
```

## Phase 3: Prioritize 用

### Must を炙り出す

```
「もし開発期間が半分になったら、何を残しますか？」

「これがないと絶対に使い物にならない、
 という機能はどれですか？」

「ユーザーが一番よく使う機能はどれですか？」
```

### 依存関係の確認

```
「[機能A] と [機能B] はどちらが先に必要ですか？
 [機能B] は [機能A] がないと動きませんか？」

「ユーザー登録なしで使える範囲はどこまでですか？」
```

### 最終確認

```
「この機能リストと優先順位で進めます。
 最後に確認ですが:
 ・漏れている機能はないか
 ・優先順位は正しいか
 ・やらないことリストに異論はないか」
```

## 対話のアンチパターン

| やりがち | 問題 | 改善 |
|---|---|---|
| 一度に5つ以上質問 | 圧倒される | 1-2個に絞る |
| 「他にありますか？」連発 | 出尽くした後も圧迫 | 選択肢を提示して選ばせる |
| 技術用語の羅列 | 非エンジニアを置き去り | 平易な言葉で言い換え |
| いきなり詳細仕様の話 | 全体像が見えない | 大枠→詳細の順で |
| 実装方法の議論に流れる | このスキルの範囲外 | 「まず何を作るか決めましょう」 |

---

## ユーザータイプ別の対応

**アイデアが発散するユーザー**:
- 「面白いですね。今回のスコープに入れるのはどれですか？」
- 定期的にスコープを確認し膨張を防ぐ

**回答が短いユーザー**:
- 具体例を示して「こういうこと？」と確認
- 選択肢を提示して選んでもらう

**技術に詳しいユーザー**:
- 実装の話に流れがちなので「何を作るか」に引き戻す
- ただし技術制約が機能に影響する場合は早期に議論

**ドメイン知識が豊富なユーザー**:
- 暗黙の前提を言語化することに注力
- 「それは業界では当たり前でも、仕様として明文化が必要です」
