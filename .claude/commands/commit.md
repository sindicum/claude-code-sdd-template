# Git コミット

変更を分析し、Conventional Commits形式のメッセージを生成してコミットします。

## 使い方

```bash
# 自動でメッセージを生成
> /commit

# メッセージを指定
> /commit feat(auth): ログイン機能を追加
```

## フロー

### 1. 変更確認

`git status` で変更を確認する。変更がなければ「コミットする変更がありません」と報告して終了。

### 2. ブランチ確認

`git branch` で現在のブランチを確認する。

- `main` または `develop` ブランチの場合: **警告**を表示し、featureブランチの作成を提案する
- `feature/*` または `fix/*` ブランチの場合: 続行

### 3. コミット実行

**引数にコミットメッセージが指定されている場合:**

1. 指定されたメッセージと変更ファイル一覧をユーザーに提示
2. ユーザー承認後にコミット実行

**引数が指定されていない場合:**

1. `Task`ツールで `git-operator` サブエージェントをモード2（コミット）で起動
2. サブエージェントが変更を分析しメッセージを生成
3. ユーザーに提示し、承認後にコミット実行

### 4. 結果報告

コミット完了後、以下を報告:

```
コミットが完了しました。

ブランチ: feature/xxx
コミット: abc1234
メッセージ: feat(scope): 説明

※ pushは手動で行ってください: git push origin feature/xxx
```
