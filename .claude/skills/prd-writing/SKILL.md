# PRD作成スキル

このスキルは、仕様駆動開発(SDD)に必要なプロダクト要求定義書(PRD)を作成するためのガイドです。

## PRDの目的

PRDは「何を作るか」を定義するドキュメントです。

**含めるべきもの:**
- プロダクトの概要と目的
- 前提条件・制約条件
- 機能要件（MVP/将来）
- 非機能要件
- スコープ外

**含めないもの（別ドキュメントの領域）:**
- 詳細なペルソナ定義（プロダクトマネジメント）
- KPI・成功指標（ビジネス指標）
- 技術スタック（アーキテクチャ設計書）
- 受け入れ条件の詳細（機能設計書/テスト仕様）

## 出力先

```
docs/product-requirements.md
```

## テンプレート

PRDを作成する際は、次のテンプレートを使用してください: ./template.md

## PRD作成プロセス

### 1. インプットの確認

`docs/ideas/` にファイルがあれば参照する。なければ対話でヒアリング。

### 2. PRDドラフトの生成

テンプレートに従ってPRDを生成する。

### 3. レビュー観点

1. 目的が明確か
2. 機能要件が実装可能なレベルで記述されているか
3. 非機能要件が測定可能か
4. スコープ外が明確か

## 記述の原則

### 具体性と測定可能性

**悪い例:**
- システムは高速である必要がある

**良い例:**
- LCP: 2.5秒以内

### 優先順位の明確化

- **P0(必須)**: MVPに含める機能
- **P1(重要)**: 初期リリース後に追加
- **P2(できれば)**: 将来的に検討

## 既存ドキュメントがある場合

`docs/product-requirements.md` が既に存在する場合は、既存の構造と内容を維持しながら更新する。
